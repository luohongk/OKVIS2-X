🎯 KEY PROBLEM IDENTIFIED
═════════════════════════════════════════════════════════════════════════════════

When Deep Learning (DL) features are enabled:
  
  ✓ SuperPoint extracts float32 descriptors (20-50ms)
  ✗ BRISK then re-scans image, extracts binary descriptors (30-100ms) ← REDUNDANT!
  
  Result: 50-150ms/frame WASTED on redundant extraction (25-40% of frontend time)

Root Cause: DL features (float, 256-dim) incompatible with BRISK DBoW (binary, 48B)
           → Forces secondary BRISK extraction for place recognition
